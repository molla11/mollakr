C/C++ 알고리즘 문제해결 환경 설정

Windows에서 WSL Ubuntu를 설치하고, VS Code에서 C/C++ 알고리즘 문제를 풀기 위한 환경 설정이다. 실행과 디버깅은 CodeLLDB와 F5를 사용한다.

1. Windows 기능 설정

  • Windows 검색창에서 Windows 기능 켜기/끄기를 실행한다.
  • 아래 항목을 체크한다.
    • Linux용 Windows 하위 시스템
    • Virtual Machine Platform 또는 가상 머신 플랫폼
    • Windows 하이퍼바이저 플랫폼
    • Hyper-V는 없을 수 있음
  • 컴퓨터를 재시작한다.

2. Ubuntu 설치

  • Microsoft Store에서 Ubuntu 24.04.1 LTS를 설치한다.
  • Ubuntu를 실행한 뒤 username과 password를 설정한다.
  • password는 입력해도 화면에 보이지 않는다.

WslRegisterDistribution failed with error: 0x800701bc 오류가 나오면 아래 파일을 설치한다.

  • wsl_update_x64.msi

3. 터미널 열기

다음 중 하나의 방법으로 wsl 터미널을 연다.
  • Windows 키를 누르고 wsl을 검색해 이를 실행한다.
  • Windows 키를 누르고 Powershell을 검색해 이를 실행하고, wsl을 입력해 wsl로 이동한다.

4. 필수 패키지 설치

sudo apt update
sudo apt install gcc g++ build-essential gdb lldb -y
whereis gcc
whereis g++
whereis gdb
whereis lldb

/usr/bin/g++처럼 경로가 나오면 된다.

5. VS Code 설치 및 실행

  • Windows에 VS Code를 설치한다.
  • Ubuntu 터미널에서 작업 폴더를 만든다.
mkdir -p ~/ps
cd ~/ps
code .

VS Code 왼쪽 아래에 WSL: Ubuntu가 보이면 된다.

6. VS Code 확장 프로그램 설치

WSL: Ubuntu 상태에서 확장 프로그램을 설치한다.

  • Ctrl + Shift + X로 확장 프로그램 탭을 연다.
  • 아래 확장 프로그램을 설치한다.
    • C/C++
    • CodeLLDB
    • Remote Development
    • Competitive Programming Helper — 선택
    • Korean Language Pack — 선택

확장 프로그램은 WSL 쪽에 설치한다.

7. 기본 터미널 설정

  • Ctrl + `로 터미널을 연다.
  • 터미널 오른쪽 위의 아래 화살표를 누른다.
  • Select Default Profile을 선택한다.
  • Ubuntu 또는 WSL을 선택한다.
  • Ctrl + Shift + P → Reload Window

8. CodeLLDB 설정

프로젝트 폴더에 .vscode 폴더를 만들고, tasks.json, launch.json을 작성한다.

.vscode/tasks.json

{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "C++ build active file",
            "type": "shell",
            "command": "/usr/bin/g++",
            "args": [
                "-std=c++17",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}

.vscode/launch.json

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "CodeLLDB: Debug active file",
            "type": "lldb",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "cwd": "${fileDirname}",
            "terminal": "integrated",
            "stdio": null,
            "preLaunchTask": "C++ build active file"
        }
    ]
}

label과 preLaunchTask 이름이 같아야 한다.

9. C++ 테스트

hello.cpp 파일을 만들고 아래 코드를 작성한다.

#include <iostream>
using namespace std;

int main() {
    cin.tie(0);
    ios::sync_with_stdio(0);

    cout << "Hello world\n";
    return 0;
}
  • F5를 누른다.
  • 처음 실행 시 CodeLLDB: Debug active file을 선택한다.
  • 터미널에 Hello world가 출력되면 된다.

10. 디버깅 방법

  • 줄 번호 왼쪽을 클릭하면 breakpoint를 찍을 수 있다.
  • F5를 누르면 실행 또는 디버깅이 시작된다.
  • 입력은 TERMINAL 탭에 한다.
  • 변수는 VARIABLES, WATCH에서 확인한다.

11. 코드 포매팅

  • Ctrl + ,로 설정을 연다.
  • Clang_format_fallback Style에 아래 값을 입력한다.
{ BasedOnStyle: Google, UseTab: Never, IndentWidth: 4, TabWidth: 4, ColumnLimit: 0 }

Shift + Alt + F로 포매팅한다.

12. 기타

  • VSCode 설정(Ctrl + ,)에서 Enable Preview를 끄면 파일이 항상 새 탭으로 열린다.
  • CPH를 설치했다면 A+B 문제로 테스트한다.
  • 필요 없는 실행 파일은 삭제한다.
rm hello

설정이 꼬이면 프로젝트 폴더의 .vscode 폴더를 삭제하고 다시 만든다.
