C/C++ 알고리즘 문제해결 환경 설정

Windows에서 WSL Ubuntu를 설치하고, VS Code에서 C/C++ 알고리즘 문제를 풀 수 있도록 환경을 맞추는 과정이다. 디버깅은 기본 GDB 대신 CodeLLDB를 사용한다.

1. Windows 기능 설정

  • Windows 검색창에서 Windows 기능 켜기/끄기를 실행한다.
  • 아래 항목들을 체크한다.
    • Linux용 Windows 하위 시스템
    • Virtual Machine Platform 또는 가상 머신 플랫폼
    • Windows 하이퍼바이저 플랫폼
    • Hyper-V는 컴퓨터에 따라 없을 수 있음
  • 설정이 끝나면 컴퓨터를 재시작한다.

2. Ubuntu 설치

  • Microsoft Store에서 Ubuntu 24.04.1 LTS를 설치한다.
  • Ubuntu를 실행한 뒤 username과 password를 설정한다.
  • password는 입력해도 화면에 보이지 않는다. 그냥 입력하고 Enter를 누르면 된다.

만약 설치 중에 WslRegisterDistribution failed with error: 0x800701bc 같은 오류가 나오면, 아래 파일을 설치한 뒤 다시 진행한다.

  • wsl_update_x64.msi

3. 터미널 설정

  • Windows Terminal 또는 Ubuntu 터미널을 실행한다.
  • 자주 쓸 거라면 Ubuntu 프로필을 기본값으로 설정해 둔다.
  • 커서 모양, 글자 크기, 폰트는 편한 대로 바꾸면 된다.

4. 필수 패키지 설치

Ubuntu 터미널에서 아래 명령어를 한 줄씩 실행한다.

sudo apt update
sudo apt install gcc g++ build-essential gdb lldb -y
whereis gcc
whereis g++
whereis gdb
whereis lldb

whereis 명령어를 실행했을 때 각 프로그램의 경로가 나오면 설치된 것이다. 예를 들어 /usr/bin/g++ 같은 경로가 보이면 된다.

5. VS Code 설치 및 WSL에서 실행

  • Windows에 VS Code를 설치한다.
  • Ubuntu 터미널에서 알고리즘 문제를 저장할 폴더를 만든다.
mkdir -p ~/ps
cd ~/ps
code .

VS Code는 가능하면 Windows에서 직접 폴더를 여는 것보다, WSL 터미널에서 code .로 여는 것이 좋다.

제대로 열렸다면 VS Code 왼쪽 아래에 WSL: Ubuntu가 보인다. 이 표시가 없으면 Windows 쪽 VS Code로 열린 것이므로 다시 WSL 터미널에서 code .를 실행한다.

6. VS Code 확장 프로그램 설치

VS Code가 WSL: Ubuntu 상태로 열린 것을 확인한 뒤 확장 프로그램을 설치한다.

  • Ctrl + Shift + X로 확장 프로그램 탭을 연다.
  • 아래 확장 프로그램을 설치한다.
    • C/C++
    • CodeLLDB
    • Remote Development
    • Competitive Programming Helper — 선택 사항
    • Korean Language Pack — 선택 사항

확장 프로그램은 Windows 로컬이 아니라 WSL 쪽에 설치되어야 한다. 확장 프로그램 버튼에 Install in WSL: Ubuntu처럼 표시되는지 확인한다.

7. 기본 터미널 설정

  • Ctrl + `로 터미널을 연다.
  • 터미널 오른쪽 위의 아래 화살표 버튼을 누른다.
  • Select Default Profile을 선택한다.
  • Ubuntu 또는 WSL을 기본 터미널로 설정한다.
  • Ctrl + Shift + P를 누르고 Reload Window를 실행한다.

8. C++ 테스트 코드 작성

hello.cpp 파일을 만들고 아래 코드를 작성한다.

#include <iostream>
using namespace std;

int main() {
    cin.tie(0);
    ios::sync_with_stdio(0);

    cout << "Hello world\n";
    return 0;
}

터미널에서 직접 컴파일하고 실행해 본다.

g++ -std=c++17 -g hello.cpp -o hello
./hello

정상적으로 설정됐다면 아래처럼 출력된다.

Hello world

9. CodeLLDB 디버깅 설정

프로젝트 폴더 안에 .vscode 폴더를 만들고, 그 안에 tasks.json과 launch.json을 만든다.

.vscode/tasks.json

{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "C++ build active file",
            "type": "shell",
            "command": "/usr/bin/g++",
            "args": [
                "-std=c++17",
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "options": {
                "cwd": "${fileDirname}"
            },
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}

.vscode/launch.json

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "CodeLLDB: Debug active file",
            "type": "lldb",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "cwd": "${fileDirname}",
            "terminal": "integrated",
            "stdio": null,
            "preLaunchTask": "C++ build active file"
        }
    ]
}

여기서 tasks.json의 label과 launch.json의 preLaunchTask 이름이 같아야 한다.

  • 코드 왼쪽 줄 번호 부분을 클릭하면 breakpoint를 찍을 수 있다.
  • F5를 누르면 디버깅이 시작된다.
  • 입력은 DEBUG CONSOLE이 아니라 TERMINAL 탭에 입력한다.
  • 변수 값은 왼쪽의 VARIABLES 또는 WATCH에서 확인할 수 있다.

10. input.txt로 디버깅하기

알고리즘 문제를 풀 때는 입력을 매번 직접 치기 귀찮으므로, input.txt를 만들어 두고 사용하는 방법도 괜찮다.

예를 들어 같은 폴더에 아래처럼 파일이 있다고 하자.

main.cpp
input.txt

이 경우 launch.json을 아래처럼 바꾸면 input.txt가 표준 입력으로 들어간다. 

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "CodeLLDB: Debug active file with input.txt",
            "type": "lldb",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "cwd": "${fileDirname}",
            "terminal": "integrated",
            "stdio": [
                "input.txt",
                null,
                null
            ],
            "preLaunchTask": "C++ build active file"
        }
    ]
}

stdio의 첫 번째 값이 입력 파일이다. 뒤의 null 두 개는 출력과 에러를 기본 터미널로 보내겠다는 뜻이다.

11. 코드 포매팅 설정

저장할 때마다 자동으로 코드가 정리되면 불편할 수 있으므로, 자동 포매팅은 꺼 두고 필요할 때만 수동으로 실행한다.

  • Ctrl + ,로 설정을 연다.
  • Format On Save를 검색한다.
  • Format On Save 체크를 해제한다.
  • Clang_format_fallback Style에 아래 값을 입력한다.
{ BasedOnStyle: Google, UseTab: Never, IndentWidth: 4, TabWidth: 4, ColumnLimit: 0 }

코드 정리가 필요할 때는 Shift + Alt + F를 누르면 된다.

12. 일반 실행

디버깅이 필요 없고 그냥 실행만 해 보고 싶다면 터미널에서 직접 실행하는 게 제일 단순하다. 

g++ -std=c++17 -g main.cpp -o main
./main

VS Code에서 빌드 작업을 쓰고 싶다면 Ctrl + Shift + B를 누르면 된다. 다만 알고리즘 문제를 풀 때는 터미널에서 직접 실행하는 방식이 가장 덜 꼬인다.

13. 기타

  • VS Code 설정에서 Enable Preview를 끄면 파일이 항상 새 탭으로 열린다.
  • CPH를 설치했다면 A+B 문제로 예제 실행이 잘 되는지 확인해 본다.
  • 필요 없는 실행 파일은 직접 삭제한다.
rm hello

설정이 이상하게 꼬였을 때는 프로젝트 폴더의 .vscode 폴더를 삭제하고 다시 만들면 된다.
